textarea: A Basic Intro

The textarea tag is a long, multi-line text input element and is useful on any website a requiring multiline text input. The textarea element is often seen as a text input to write comments on a blog or forum, to write reviews, to give feedback, and/or for long text input for various web apps. It is able to hold a virtually unlimited amount of text and by default displays text using a fix width font.

textarea: Attribues and Settings

A textarea is a type of input, but not a type of input element like other common input elements. It requires an opening an closing tag as such:

<textarea rows="10" cols="50"> This is a textarea. </textarea>

Output:

The text between the tags will display in the textarea. This text is not a placeholder text, and if not modified the default text will be sent as form data. If you wish to set placeholder text, it is best to use the placeholder attribute as such:

<textarea rows="10" cols="50" placeholder="This is a textarea placeholder"></textarea>

Output:

Note that there can be no spaces between the textarea tags as space will be interpreted as default text, thus negating the placeholder text.

You may have also noticed the row and column attributes, which control the number of rows and columns the textarea will display.

The size of the text area can also be controlled via CSS using the width and height properties. It is also important to note that the row and column attributes do no control the amount of possible input. Even a 1 x 1 textarea can accept an unlimited amount of text. To control the actual amount of input, simply enter the maximum amount of characters you wish to accept using the maxlength attribute as so:

<textarea rows="10" cols="50" placeholder="Try entering more than 10 characters! I DARE YA!" maxlength="10"></textarea></textarea>

Try to enter more than ten characters in the textarea above. As you will see it is impossible. Even if you paste text into the textarea, it will cut off the text longer than 10 characters.

Another important attribute commonly seen with the textarea element is the wrap attribute. A text line wrap is when text automatically jumps to a new line when a user enters text too long for the current line. The textarea wrap attribute declares how to display these wraps when editing, and how to send textarea‘s line wrapping data to the server. The soft value, which is the default, still permits text wrapping when editing in a textarea, but ignores any new lines not user inputted when sending textarea text data (new lines created by automatic line wraps are ignored and not sent). The hard setting also displays line wraps when editing but sends all new lines created by text wrapping in the textarea text data to the server. The off value turns text wrapping off and sends text data to the server exactly as the user entered.

Here is an example of a textarea with the wrap attribute set to off:

HTML:

<textarea rows="10" cols="50" wrap="off">This is a lot of text you know, so please be careful with the text wrapping attribute. </textarea>

Output:

And here is the same textarea with the wrap attribute set to soft:

HTML:

<textarea rows="10" cols="50" wrap="soft">This is a lot of text you know, so please be careful with the text wrapping attribute. </textarea>

Output:

Note how the textbox scrolls horizontally when wrap is set to off. Also, note that a textarea col element must be set when setting wrap to hard.

Another important attribute is the readonly attribute, allowing the textarea to display input, but reject any input, which is quite useful for any web page requiring a text area to display output. Here is a brief example:

HTML:

<textarea rows="10" cols="50" readonly="true">This text area is for display purposes only. </textarea>

Output:

Note that the text is still selectable.

textarea also can take attributes common to other form input elements such as autofocus, disabled, required, form, name etc.
